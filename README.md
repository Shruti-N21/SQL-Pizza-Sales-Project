🍕 Pizza Sales Optimization – SQL & Business Analysis Case Study
📌 Project Overview

This project demonstrates an end-to-end Business Analyst approach combined with SQL data analysis to optimize pizza sales performance for a restaurant business.

The analysis focuses on identifying sales inefficiencies, understanding product performance, and recommending data-driven strategies to improve revenue and operational utilization.

🎯 Business Problem

The restaurant faced multiple operational challenges:

Uneven sales distribution across time slots

Heavy dependency on lunch and dinner hours

Low-performing XXL size and Chicken category pizzas

Inventory wastage during non-peak hours

Generic promotions without data-backed insights

Staff underutilization during slow periods

🧠 Project Objective

To analyze pizza sales data using SQL and propose business solutions that:

Increase non-peak hour revenue

Improve low-performing product categories

Reduce inventory wastage

Enable data-driven decision-making

📊 Data Analysis Approach (SQL)

Sales data was analyzed using SQL techniques including:

JOIN

GROUP BY

Aggregate Functions (SUM, COUNT)

Time-based sales analysis

Category & size performance analysis

🔍 Key Insights

Peak sales occur between 12 PM–3 PM and 7 PM–10 PM

XXL pizza size shows lowest demand

Chicken category underperforms despite high revenue potential

Revenue concentration exists in limited products and time windows
